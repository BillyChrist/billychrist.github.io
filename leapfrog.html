LEAPFROG

Lunar Entry and Approach Platform for Research on Ground

LEAPFROG is a full-scale autonomous lunar lander test platform developed at USC’s Space Engineering Research Center to support hardware-in-the-loop testing of guidance, navigation, and control systems.

My work focused on project leadership, system design and architecture, vehicle integration and testing, mechanical design, and development of embedded flight software for closed-loop control.

System Overview

Project Details and Involvement

  • Development of the vehicle required resolving several system-level design challenges spanning structures, propulsion, flight computing, and control system stability.
  • The landing leg assembly was redesigned to support repeated landings and survive a 1 meter drop test. Structural analysis and redesign of the suspension interface increased the survivable landing impulse while maintaining strict vehicle mass limits.
  • The original cold-gas reaction control system was replaced with an electrically driven attitude control system using eight electric ducted fans powered by rechargeable LiPo batteries. This eliminated reliance on consumables and enabled repeatable test operations without external gas infrastructure.
  • The flight computing architecture was upgraded from a single-board control solution to a distributed architecture using an STM32 microcontroller for real-time control tasks and a Raspberry Pi for high-level coordination, enabling RTOS-based control loops and ROS2 integration.
  • A control strategy was developed to resolve coupling between the turbine thrust vector control system and the attitude control system, preventing oscillatory behavior and allowing the two control systems to cooperate during stabilization.
  • Vehicle mass growth analysis showed that it exceeded the propulsion system lift margin. Component-level mass optimization and material selection were performed across the vehicle to maintain flight capability while preserving structural and functional requirements.
  • I led the integration of these systems, bringing the project from a proposed component list to a functional experimental flight vehicle through mechanical design, embedded software development, and system integration testing.

Electronics

  • Integrated avionics stack (STM32 + Raspberry Pi) with sensor suite, radios, and power distribution.
  • Developed wiring and interface documentation to support rapid assembly and test iteration.

Mechanical

  • Redesigned leg mounts to secure the swivel joint without adhesives or welding, improving strength and rigidity.
  • Performed impulse force calculations for worst-case landing (34 kg vehicle, 10 m/s, 6800 N impact force).
  • Ran FEA to estimate deformation and stress; iterated bracket geometry to survive worst-case loading with acceptable strain.

Attitude Control System (ACS)

  • Implemented PID-based control laws for pitch, roll, and yaw using fan pairs (PR1–4 and Y1–2).
  • Tuned controller gains based on full ACS test output graphs to achieve stable response.
  • Normalized post-calculation PID outputs mapped directly to ESC commands.

Flight Software (FSW)

  • Dual-layer architecture: STM32 with RTOS for low-level control and sensor fusion; Raspberry Pi with ROS2 for high-level mission logic.
  • CLI-based ground station executing customizable flight scripts with telemetry logging.
  • Heartbeat-based command chain (CLI → Pi → STM32) with ROS bridge nodes for telemetry and safety behaviors (safe landing, engine cooldown on heartbeat loss).
