LEAPFROG

Lunar Entry and Approach Platform for Research on Ground

SERC project focused on a reusable lunar-lander style testbed for guidance, navigation, and control research. Work spanned electronics integration, mechanical design, and flight software architecture.

System Overview

Multirotor-like test vehicle with jet engine, electric ducted fans, and ground station link for scripted flight tests.

  • Component stack including GST, RFD 900X radios, Raspberry Pi 4, STM32, ESCs & EDFs, multiple IMUs, LIDAR, linear actuators, and JetCat Pro engine.
  • Used for validating attitude control, state estimation, and autonomous landing behaviors.

Electronics

  • Integrated avionics stack (STM32 + Raspberry Pi) with sensor suite, radios, and power distribution.
  • Developed wiring and interface documentation to support rapid assembly and test iteration.

Mechanical

  • Redesigned leg mounts to secure the swivel joint without adhesives or welding, improving strength and rigidity.
  • Performed impulse force calculations for worst-case landing (34 kg vehicle, 10 m/s, 6800 N impact force).
  • Ran FEA to estimate deformation and stress; iterated bracket geometry to survive worst-case loading with acceptable strain.

Attitude Control System (ACS)

  • Implemented PID-based control laws for pitch, roll, and yaw using fan pairs (PR1–4 and Y1–2).
  • Tuned controller gains based on full ACS test output graphs to achieve stable response.
  • Normalized post-calculation PID outputs mapped directly to ESC commands.

Flight Software (FSW)

  • Dual-layer architecture: STM32 with RTOS for low-level control and sensor fusion; Raspberry Pi with ROS2 for high-level mission logic.
  • CLI-based ground station executing customizable flight scripts with telemetry logging.
  • Heartbeat-based command chain (CLI → Pi → STM32) with ROS bridge nodes for telemetry and safety behaviors (safe landing, engine cooldown on heartbeat loss).
